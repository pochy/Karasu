# Tauri 超軽量 Markdown ビューワー 設計書

> **実装後の設計判断（候補・理由・スコープ変更）**は [`docs/design-decisions.md`](../../design-decisions.md) にまとめています。  
> 本ファイルはプロジェクト開始時点の要件定義です。

## 目的

Tauri で、起動が速く、依存が少なく、Markdown の閲覧と編集に集中したデスクトップアプリを作る。

対象は「Markdown ファイルを開いて編集し、明示保存する」用途に絞る。余計な機能は追加しない。

## スコープ

### 含めるもの

- Markdown ファイルを開く
- Markdown を編集する
- Markdown をプレビュー表示する
- `Ctrl/Cmd+S` による明示保存
- 最近開いたファイルの復元
- 編集画面とプレビュー画面の切り替え

### 含めないもの

- リアルタイム共同編集
- WYSIWYG エディタ
- Git 連携
- 同期サービス
- 複数ペインの常時表示
- ノート管理機能

## 体験方針

アプリは単一ウィンドウで動作し、既定では編集画面を表示する。プレビューは同じウィンドウ内で切り替えて確認する。

編集体験は軽さを優先する。高機能な編集補助よりも、起動速度、入力の素直さ、保存の確実性を重視する。

## 画面構成

### 編集画面

- 上部に最小限のツールバー
- 中央に Markdown のテキスト入力領域
- 下部に保存状態を示すステータス

### プレビュー画面

- Markdown をレンダリングした読み取り専用表示
- 見出し、段落、箇条書き、コード、リンク、表を見やすく整形
- 長文でも読みやすい余白と行間を確保

### 共通要素

- ファイル名表示
- 保存状態表示
- 画面切り替えボタン
- ファイルを開く操作

## 技術構成

### Rust 側

- ファイル読み込み
- ファイル保存
- 最近使ったファイルの保持
- パス解決とエラーハンドリング

### フロントエンド側

- エディタ UI
- プレビュー UI
- 表示状態管理
- キーボードショートカット

### Markdown 処理

軽量な Markdown パーサを使い、HTML への変換はフロントエンドで行う。

採用方針は次の通り。

- CommonMark 系の基本構文を優先
- コードブロック、見出し、リスト、引用、リンク、表を確実に表示
- 拡張機能は最小限に留める

## データフロー

1. ユーザーがファイルを開く
2. Rust 側が内容を読み込む
3. フロントエンドがテキストを編集状態として保持する
4. プレビュー表示時に Markdown を HTML に変換する
5. `Ctrl/Cmd+S` で Rust 側へ保存要求を送る
6. 保存成功時に保存状態を更新する
7. 起動時に最近開いたファイルがあれば復元する

## 保存仕様

保存は明示的に行う。自動保存はしない。

- `Ctrl/Cmd+S` で現在の内容を上書き保存
- 保存前に対象パスが有効か確認する
- 保存失敗時はエラーを画面に出す
- 保存成功時は「保存済み」状態を表示する

## エラー処理

想定する失敗は次の通り。

- ファイルが存在しない
- 読み取り権限がない
- 書き込み権限がない
- 保存先が不正
- Markdown 変換時に予期しない入力がある

扱い方は次の通り。

- 画面内に簡潔なエラーメッセージを表示する
- アプリを落とさず、編集状態は保持する
- 保存失敗時は未保存であることを明確に示す

## パフォーマンス方針

このアプリの価値は軽さにあるため、次を守る。

- 重い依存を増やしすぎない
- 画面の再描画を局所化する
- 大きな Markdown でも入力遅延を出さない
- プレビューは必要時に更新する
- 状態管理は最小限にする

## アクセシビリティ

- キーボードだけで開く、切り替える、保存する操作を完結させる
- フォーカス状態を明確にする
- 文字サイズと行間を読みやすく保つ
- 色だけに依存しない保存状態表示を使う

## テスト方針

### Rust 側

- ファイル読み込み成功
- ファイル読み込み失敗
- 保存成功
- 保存失敗
- 最近開いたファイルの復元

### フロントエンド側

- 編集内容の状態保持
- プレビュー切り替え
- 保存ショートカット
- エラー表示

### 手動確認

- 起動が速いこと
- 長文 Markdown でも入力が重くならないこと
- 保存後に内容が正しく反映されること
- プレビューで基本構文が崩れないこと

## 実装順序

1. Tauri の土台を作る
2. ファイル読み込み・保存を実装する
3. 編集画面を作る
4. Markdown プレビューを実装する
5. 切り替え UI とショートカットをつなぐ
6. 最近開いたファイルの復元を追加する
7. 画面全体の軽量化と調整を行う

## 成功条件

- アプリがすぐ起動する
- Markdown ファイルを開ける
- 編集できる
- `Ctrl/Cmd+S` で保存できる
- 編集とプレビューを切り替えられる
- 保存や読込で落ちない
- 余計な機能が入っていない

## 実装後に追加された機能（設計書初版の外）

次は初期スコープの後に入れたものです。詳細は [`docs/design-decisions.md`](../../design-decisions.md) を参照。

- **作業フォルダ**とファイルサイドバー（遅延ツリー、一覧の開閉）
- プレビュー DADS 風スタイル、**sugar-high** によるシンタックスハイライト
- **設定モーダル**（システムフォント列挙、文字サイズ）

「ノート管理」に相当するタグ・検索・バックリンク等は引き続き含めていません。
